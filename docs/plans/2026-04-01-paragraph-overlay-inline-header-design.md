# Paragraph Overlay Inline Header Design

Date: 2026-04-01

## 需求分析

- 当前句子翻译浮层在顶部存在两层垂直结构：
  - 第一层是仅包含 `Pin` / `Close` 的独立顶栏。
  - 第二层是正文里的“原文”标题行。
- 这会在原文正文上方形成明显留白，尤其在短句场景下更容易显得面板偏高。
- 用户期望：
  - “原文”标题与右上角控制按钮共用一行。
  - 按钮相对上边框的贴边位置保持与当前 `xmark` 一致。
  - 调整后减少顶部无效留白，但不破坏现有交互。

## UI 设计

### 方案 A：把标题行并入当前固定顶栏

- 用新的固定标题栏替代当前仅显示按钮的 `paragraphTopBar`。
- 标题栏左侧显示“原文”与复制按钮，右侧显示 `Pin` / `Close`。
- 原文正文从标题栏下方直接开始，不再在正文区域重复绘制“原文”标题。

优点：
- 保持当前按钮贴近上边框的位置。
- 保持标题栏固定，长文本滚动时 `Pin` / `Close` 仍始终可见。
- 改动范围集中在 `OverlayView.swift`，不影响窗口布局逻辑。

缺点：
- 需要为段落结果视图增加一个“原文标题栏”分支。

### 方案 B：把标题和按钮都挪进正文区域

- 删除固定顶栏。
- 在正文首行渲染“原文”标题、复制按钮和右上角控制按钮。

优点：
- 结构看起来更直接。

缺点：
- 按钮会比当前更靠下，违背“保持贴边位置”的要求。
- 在滚动场景下标题和关闭按钮会一起滚走，交互退化。

## 推荐方案

采用方案 A。

这是最小改动且最符合目标的实现方式：把现有顶部栏从“纯按钮栏”升级成“原文标题栏”，保留当前顶部贴边关系，同时消除标题重复带来的留白。

## 实施要点

1. 在段落结果态中，用新的 `paragraphOriginalTopBar` 替代现有 `paragraphTopBar`。
2. 新顶栏沿用当前顶部 `10pt` 上边距和 `6pt` 下边距，让按钮保持现有贴边位置。
3. 原文正文区域删除重复标题，只保留正文内容。
4. 其他段落状态（如 loading）继续使用现有 `paragraphTopBar`，避免扩大改动面。

## 验收标准

- 段落翻译结果中，“原文”与 `Pin` / `Close` 在同一行。
- 右上角按钮相对上边框的垂直位置与改动前一致。
- 原文正文上方留白明显减少。
- 长文本滚动时，顶部控制按钮仍固定可用。
